 Critical Files

 - scripts/rank_papers.py - Ranking algorithm
 - scripts/generate_report.py - Output formatting
 - config/search_config.yaml - Easy parameter tuning

 Verification Steps

 1. Local testing:
   - Run python scripts/fetch_papers.py to verify API connectivity
   - Check that papers are fetched and cached
   - Run python scripts/generate_report.py to verify markdown output
 2. GitHub Actions testing:
   - Use workflow_dispatch to manually trigger workflow
   - Verify that reports are generated and committed
   - Check GitHub Actions logs for errors
 3. Monthly automation:
   - Wait for scheduled run on 1st of next month
   - Verify new report appears in data/ folder
   - Compare with previous month to see citation growth
 4. Output validation:
   - Verify top papers are truly VLM/VLA related
   - Check that citation counts match Semantic Scholar website
   - Ensure markdown renders correctly on GitHub

 Trade-offs & Decisions

 - Semantic Scholar vs Google Scholar: Semantic Scholar chosen for free API access and good citation data
 - Markdown vs Dashboard: Markdown chosen for simplicity, version control, and GitHub rendering
 - Monthly vs Weekly: Monthly updates balance freshness with API rate limits and noise reduction
 - Citation count only: Simple metric; could expand to include influence scores, h-index, etc. in future
